# PRIMER: Prompt-Ready Interface for Multimodal Extraction and Representation

> **S2AI (Science to AI) 的“底漆”工程：将复杂科学数据转化为大模型可读的语义表征。**

## 1. 项目愿景：为什么需要 PRIMER？

当今主流的文本大模型（LLM）擅长处理自然语言，但在面对“硬核科学”（Hard Science）数据时力不从心。科学研究的原始形态往往是复杂的目录结构、包含高维图表的学术 PDF，以及非人类可读的二进制仿真结果。

**PRIMER** 并非要创造一个新的世界模型，而是充当科学数据与 LLM 之间的“**底漆（Primer）**”。通过对原始数据进行解构、提取和重新表征，PRIMER 抹平了异构数据间的粗糙纹理，为上层 LLM 提供了标准化的、高附着力的“提示词就绪（Prompt-Ready）”界面。

**我们要做的不是 AI4S，而是先完成 S2AI（Science to AI）。**

---

## 2. 核心功能架构

PRIMER 提供了三层级的数据转译能力，确保 LLM 能够真正“读懂”科研项目：

### 📁 1. 项目文件夹语义摘要 (Directory Digest)
将包含人类可读文档、机器可读配置及二进制文件的项目目录，处理为结构化的文本摘要。
- **元数据分析**：自动识别文件树层级，提取关键元数据。
- **字符编码处理**：智能识别并统一处理各类科学计算源文件的编码。
- **语义路由**：根据文件扩展名和内容特征，自动选择后续的处理策略。

### 📄 2. 科学文献多维解析 (Scientific PDF Parser)
针对学术论文 PDF，突破简单的文本提取，实现“由粗到细”的逻辑转换。
- **分层提取**：不仅是文字，更侧重于将科学图表（直方图、曲线、等高线图）转化为文本逻辑描述。
- **长表重建**：将超长数据表转换为 LLM 易于理解的 Markdown 或 JSON 结构。
- **按需细化**：支持根据 Prompt 需求，对图表中的特定细节进行更深度的数值化描述。

### 🔢 3. 二进制数据转译器 (Binary Data Translator)
将阶段性的模拟仿真结果或数据库二进制文件转化为具备物理含义的文本信息。
- **数值表征**：将 CFD 仿真、轨道星历计算等输出的二进制矩阵转化为关键特征描述。
- **模式提取**：识别数值序列中的趋势、异常点或物理规律，以自然语言或结构化数据的形式作为 LLM 输入。

---

## 3. 技术路线：S2AI 工作流

1. **输入层**：杂乱的科研项目文件夹（包含 `.pdf`, `.bin`, `.cfg`, `.py`, `.cpp` 等）。
2. **提取层 (Extraction)**：
   - 文本段落语义对齐。
   - 图像特征到文本逻辑的映射（Image-to-Logic）。
   - 二进制数值到语义摘要的压缩（Binary-to-Semantic）。
3. **表征层 (Representation)**：生成高度结构化的 JSON/Markdown。
4. **输出层 (Prompt-Ready)**：作为标准化的 Context 输入给纯文本 LLM。
